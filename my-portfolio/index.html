Executive Summary

The SIF-402 AR Station Monitor demonstrates the complete pipeline from OPC UA industrial protocols through REST APIs to immersive AR visualization on the Microsoft HoloLens 2. The application renders real-time PLC telemetry data (hopper levels, station status, alarms) onto a spatial UI in the operator's field of view, enabling hands-free machinery monitoring.

Contribution 2

Modular Data-UI Architecture

Created a flexible three-layer architecture to switch between Mock Data (Dev) and Live API (Prod) without changing UI code.

  • Layer 1: Shared Data Structures (DataClasses.cs).
  • Layer 2: Swappable Providers (MockDataManager vs ApiDataManager).
  • Layer 3: Event-Driven Presentation (StationInfoDisplay).
C# - StationInfoDisplay.cs
void OnDataReceived(StationData data) {
    if (data.station.status == "Running")
        stationStatusText.color = Color.green;
    
    // Update hoppers dynamically
    UpdateHopperDisplay(blueHopper, data.hoppers[0]);
}

void UpdateHopperDisplay(TMP_Text text, HopperData h) {
    text.text = $"Level: {h.level}%";
    if (h.status == "Low") text.color = Color.red;
}

Problem Solving Showcase

7 critical blocking issues resolved during development.

1. The 2D Window Mystery

App appeared as a flat slate, not 3D holograms.
Solution: Switch Build Type to "D3D Project".

2. OpenXR Build Failure

Error: "Holographic Remoting and Init XR both enabled".
Solution: Disable Holographic Remoting in Project Settings.

3. HTTP Security Block

Unity blocked Node-RED connection.
Solution: Player Settings -> Allow Downloads over HTTP.

4. The Invisible UI

Data fetched but UI showed "NULL".
Solution: Added missing StationInfoDisplay component.

5. Dead Hand Inputs

Buttons worked in Editor, failed on Device.
Solution: Added Microsoft Hand Interaction Profile.

6. Network Timeout

Access Denied on Local Network.
Solution: Enable "PrivateNetworkClientServer" Capability.
